# Delta for Cyberpunk Install

## MODIFIED Requirements

### Requirement: Plugin Component Install

The system SHALL copy a plugin file into `~/.config/opencode/plugins/cyberpunk.ts`. The plugin file MUST reference sound files with the `.wav` extension, matching the files generated by the Sounds Component Install. If the file already exists and is identical, it MUST report `skipped`.

(Previously: Plugin install copied the file but did not specify sound file extension consistency with the sounds component.)

#### Scenario: Fresh plugin install

- GIVEN `~/.config/opencode/plugins/cyberpunk.ts` does not exist
- WHEN `cyberpunk install --plugin` is run
- THEN the file is copied and `InstallResult.status` is `"success"`
- AND all sound file references in the plugin use `.wav` extension

#### Scenario: Plugin already installed (identical)

- GIVEN `~/.config/opencode/plugins/cyberpunk.ts` exists and matches the source
- WHEN `cyberpunk install --plugin` is run
- THEN `InstallResult.status` is `"skipped"` and no file is overwritten

#### Scenario: Plugin plays sounds with correct extension

- GIVEN the plugin is installed and sounds are installed
- WHEN a session event fires (idle, error, compacted, permission)
- THEN the plugin calls `playSound()` with the `.wav` file path
- AND `playSound()` finds the file via `existsSync()` and plays it

#### Scenario: Sound file not found is silently handled

- GIVEN the plugin is installed but sounds are not installed
- WHEN a session event fires
- THEN the plugin calls `playSound()` with the `.wav` file path
- AND `existsSync()` returns false, and the try/catch silently swallows the error

## ADDED Requirements

### Requirement: Sound Extension Consistency

The distributed plugin file (`cyberpunk-plugin.ts`) SHALL use the `.wav` file extension for all four sound file references. The sound names MUST be: `idle.wav`, `error.wav`, `compact.wav`, `permission.wav`. These MUST exactly match the file names produced by the sounds component (`src/components/sounds.ts`).

#### Scenario: All four sound references use .wav

- GIVEN the file `cyberpunk-plugin.ts` in the project root
- WHEN the file is inspected
- THEN it contains `playSound($, "idle.wav")` at the idle event handler
- AND it contains `playSound($, "error.wav")` at the error event handler
- AND it contains `playSound($, "compact.wav")` at the compacted event handler
- AND it contains `playSound($, "permission.wav")` at the permission event handler

#### Scenario: No .m4a references remain

- GIVEN the file `cyberpunk-plugin.ts` in the project root
- WHEN the file is inspected
- THEN it contains zero occurrences of the string `.m4a`
